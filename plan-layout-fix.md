# Plan: 优化页面布局

## 问题分析

当前布局存在以下问题：

1. **标题栏仅在右侧主区域**：左侧 sidebar 顶部无法拖动窗口
2. **窗口控制按钮位置不对**：嵌在主聊天区域内部，没有在窗口最右上角
3. **视觉不统一**：sidebar 顶部和主区域标题栏没有对齐，高度不一致

## 目标布局

```
┌─────────────────────────────────────────────[─ □ ✕]┐
│  (全宽标题栏，可拖动，h-10)                         │
├──────────┬─────────────────────────────────────────┤
│          │                                         │
│ + New    │                                         │
│ Chat     │       Chat Messages                     │
│          │       (flex-1, overflow-y-auto)          │
│ Session  │                                         │
│ List     │                                         │
│          │                                         │
│ (w-64,   ├─────────────────────────────────────────┤
│ 占满剩余 │  Input Area (h-20, fixed)               │
│ 高度)    │  [input field]            [Send]         │
│          │                                         │
└──────────┴─────────────────────────────────────────┘
```

## 修改方案

### 修改文件：`src/App.tsx`

将整体结构从「左右分栏」改为「上下 + 左右」三段式：

1. **最外层容器**：`flex flex-col h-screen`（纵向排列）

2. **第一行：全宽标题栏**（h-10，flex-shrink-0）
   - `-webkit-app-region: drag` 使整个标题栏可拖动
   - 右侧放置最小化、最大化、关闭按钮，设置 `no-drag`

3. **第二行：内容区域**（flex-1，flex 横向）
   - **左侧 Sidebar**（w-64，min-w-64，flex flex-col，全高）
     - "+ New Chat" 按钮
     - session 列表（flex-1，overflow-y-auto）
   - **右侧主区域**（flex-1，flex flex-col）
     - 聊天消息区（flex-1，overflow-y-auto）
     - 输入框区域（h-20，flex-shrink-0，固定在底部）

- 窗口控制按钮在右上角，功能正常

- 聊天消息区填满剩余空间
- 输入框固定在底部
